Honeynet Research Platform

Apache Guacamole Active Directory ELK Stack Security Onion MISP OpenCTI Ghidra Palo Alto PA-3410 Juniper EX4300

From-scratch honeynet built to simulate a vulnerable agricultural IoT startup. Captured live malware including a Mirai botnet variant and a crypto miner within hours of going live. Full detection stack, real adversary TTPs, documented lessons learned.

This project was built entirely from the ground up as a capstone at Pima Community College. No templates, no T-Pot scripts. Everything was planned, built, and operated manually across every layer of the stack.

The concept was to simulate an agricultural IoT startup that appeared to have accidentally pre-exposed infrastructure. A full company identity was created including a public-facing website (agricultureintech.net), six employee personas with seeded credentials pulled from known breach datasets and common wordlists including RockYou, and a convincing web presence targeting what is genuinely an underserved and high-value attack surface for nation-state actors.

Front end: Apache web server with TLS/SSL, proper DNS and CNAME records, Apache Guacamole running on Tomcat as a deceptive RDP portal. Back end: Windows Server 2019 domain controller with two Windows 11 workstations. CVE selection focused on the past two years of disclosed vulnerabilities. Final selections included an older ActiveMQ instance with a known RCE vulnerability, Jenkins, and Metabase. The environment received its first malware within six hours of going live.

Initial architecture containerized everything through Portainer including Elasticsearch, Kibana, Logstash, OpenCTI, MISP, and Arkime in separate containers. This created cascading stability issues. After troubleshooting, the team migrated to Security Onion as the primary SIEM and detection platform, with OpenCTI running in a standalone Debian VM on systemd. That was the right call and one of the key lessons from the project.

Hardware: Dell PowerEdge R6515, Palo Alto PA-3410 firewall, Juniper EX4300 switch.

Malware captured: RondoDox (ELF64, statically linked, musl libc, 9-layer custom XOR encryption, 15+ DDoS attack vectors, SSH propagation, competition killer that terminates rival malware, anti-analysis targeting 16+ security tools) and Kinsing (5.9MB Go binary, 3,826 functions, full C2 infrastructure, deploys XMRig miner kdevtmpfsi that ran at 401% CPU). Full static and dynamic analysis performed using Ghidra 11.3.1, objdump, readelf, and strings. Both binaries fully documented with SHA-256 hashes, IOCs, and MITRE ATT&CK mappings.

Theory on infection chain: RondoDox likely acted as a loader-as-a-service entry point. Initial infection established persistence and exfiltrated the endpoint into a compromised pool. That pool was then sold or leased, and a separate operator delivered the Kinsing/XMRig payload through a second C2 server. This is consistent with documented RondoDox LaaS behavior reported by CloudSEK, Trend Micro, and FortiGuard in mid-to-late 2025.

Home Lab Infrastructure

Dell PowerEdge pfSense UniFi US-8-60W Proxmox Docker ELK

Enterprise-grade home lab built on Dell PowerEdge servers running Proxmox virtualization. Segmented network with pfSense firewalls and an ELK monitoring stack. Switching is handled by a Ubiquiti UniFi US-8-60W managed switch with VLANs configured for network segmentation. Serves as the foundation for all research, testing, and training.

Infrastructure includes multiple VMs for project testing, sandbox environments, and ongoing research, a dedicated malware analysis sandbox, Active Directory lab environment, and containerized services via Docker. Network monitoring covers all segments with Zeek and Suricata. Regular vulnerability scanning with OpenVAS. All configurations managed as code for reproducibility.

Spear Phishing / BEC Analysis

Email Headers OSINT Python MITRE ATT&CK

Analyzed real-world business email compromise (BEC) and spear-phishing campaigns. Performed header analysis, sender reputation checks, URL/domain investigation, and payload extraction. Mapped tactics to MITRE ATT&CK framework and produced incident reports.

Developed Python scripts for automated header parsing and IOC extraction. Created detection rules for common BEC patterns. Documented social engineering techniques observed in the wild and built awareness training materials based on findings.
