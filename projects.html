Honeynet Research Platform, Academic Capstone

Apache Guacamole Active Directory ELK Stack Security Onion MISP OpenCTI Ghidra Palo Alto Juniper

From-scratch honeynet deployment built to simulate a vulnerable agricultural IoT startup. Captured live malware within hours of going live. Full detection stack, real adversary behavior, real lessons learned.

This was not a script or a template deployment. This project was built entirely from the ground up as part of my capstone at Pima Community College, and it required planning and execution across every layer of the stack.

The concept: simulate a small agricultural IoT startup that appeared to have accidentally exposed pre-launch infrastructure. A full company identity was created including a public-facing website (agricultureintech.net), six employee personas, and a convincing web presence targeting an underserved and high-value attack surface for nation-state actors.

The front end consisted of an Apache web server with TLS/SSL configured, proper DNS and CNAME records, and Apache Guacamole running on Tomcat as a deceptive RDP portal. The back end ran a Windows Server 2019 domain controller with two Windows 11 workstations. Credentials were seeded from known breach data and common wordlists including RockYou to make the environment appear legitimately misconfigured.

Multiple CVEs were researched and selected for deployment, targeting the past two years of disclosed vulnerabilities. The final selections included an older ActiveMQ instance with a known RCE vulnerability alongside Jenkins and Metabase. The environment was live and receiving malware within six hours.

The initial architecture containerized everything including Elasticsearch, Kibana, Logstash, OpenCTI, MISP, and Arkime in separate containers managed through Portainer. This introduced cascading stability issues. After troubleshooting, the decision was made to migrate to Security Onion as the primary SIEM and detection platform, with OpenCTI running in a standalone Debian VM on systemd rather than containerized. This was the right call and one of the key lessons learned from the project.

Hardware: Dell PowerEdge R6515, Palo Alto PA-3410 firewall, Juniper EX4300 switch.

The project produced documented TTPs from real adversary behavior, malware samples submitted for analysis, and a formal research paper comparing observed techniques to known threat actor playbooks.

Home Lab Infrastructure

Dell PowerEdge pfSense UniFi Proxmox Docker ELK

Enterprise-grade home lab built on Dell PowerEdge servers running Proxmox virtualization. Segmented network with pfSense firewalls and an ELK monitoring stack. Switching handled by a Ubiquiti UniFi 8-port managed switch with VLANs configured for network segmentation. Serves as the foundation for all research, testing, and training.

Infrastructure includes multiple VMs for project testing, sandbox environments, and ongoing research, a dedicated malware analysis sandbox, Active Directory lab environment, and containerized services via Docker. Network monitoring covers all segments with Zeek and Suricata. Regular vulnerability scanning with OpenVAS. All configurations managed as code for reproducibility.

Spear Phishing / BEC Analysis

Email Headers OSINT Python MITRE ATT&CK

Analyzed real-world business email compromise (BEC) and spear-phishing campaigns. Performed header analysis, sender reputation checks, URL/domain investigation, and payload extraction. Mapped tactics to MITRE ATT&CK framework and produced incident reports.

Developed Python scripts for automated header parsing and IOC extraction. Created detection rules for common BEC patterns. Documented social engineering techniques observed in the wild and built awareness training materials based on findings.
